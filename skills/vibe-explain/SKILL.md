name: vibe-explain
description: 用大白话和建筑比喻向非技术用户解释代码质量问题，适用于 reporter 生成总结、解释维度含义、给出改进建议等场景
---

## 触发条件

当需要向完全不懂技术的 vibe coder 解释代码质量问题时使用本 Skill。

具体触发场景：
- reporter 生成"一句话总结"时
- 用户询问某个维度的含义时
- 给出改进建议时

---

## 核心原则

### 1. 禁止使用技术术语（除非立即解释）

❌ 不能说：
> "圈复杂度过高导致代码可测试性下降"

✅ 应该说：
> "代码逻辑太绕了，就像一个有很多岔路的迷宫，容易出 bug 也很难改"

---

### 2. 必须使用建筑比喻体系

本插件统一使用建筑比喻，让用户建立一致的心智模型。同一概念可根据语境选择不同说法：

| 代码概念 | 可选的建筑比喻 |
|----------|--------------|
| 整体代码质量 | 房子的建造质量 / 这栋楼的施工水准 |
| 复杂度高 | 结构设计很混乱，墙和柱子摆得奇怪 / 走廊太多太绕，找个房间要转好几圈 |
| 有安全漏洞（量多） | 好几扇窗户没锁，风险不小 / 这栋楼有好几处明显的安全隐患 |
| 有安全漏洞（量少） | 还有一扇窗户没锁好 / 有个小角落没注意到 |
| 安全漏洞已清零 | 把所有没锁的窗户都锁上了 |
| 代码重复多 | 同一面墙砌了好几遍 / 有些房间几乎一模一样却没合并 |
| 代码重复已消除 | 把重复砌的墙拆掉合并了 |
| 可读性差 | 没有标注每个房间是干什么的 / 没有门牌，住户自己都容易迷路 |
| 可维护性差 | 改一面墙会导致另外三面墙裂开 / 结构耦合太紧，动哪里都牵一发动全身 |
| 重构后提升 | 请了设计师重新规划了一下 / 做了一次整体翻新 |
| 某项退步 | 这里加了一道墙，空间反而更局促了 |

---

### 3. 评级必须用表情而非数字（对外显示时）

对 vibe coder 展示时，综合分对应以下评级：

| 综合分（/60） | 表情评级 | 大白话 |
|-------------|---------|--------|
| 54–60 | 🏰 | 建造扎实，可以放心住人 |
| 42–53 | 🏠 | 能住，但有些地方需要修缮 |
| 30–41 | 🏗️ | 还在施工状态，上线前需要加固 |
| 18–29 | 🔨 | 毛坯房，能看出结构但远未完成 |
| < 18  | 💩 | 需要推倒重来 |

---

### 4. 每个问题必须说"如果不管会怎样"

解释问题时必须包含后果说明，让用户理解为什么要在意：

模板：
> "[这个问题]就像[建筑比喻]。如果不管，[具体后果]。"

示例：
> "安全性得分低，就像窗户没锁好。如果不管，别人可能通过你的网站拿到用户数据，到时候麻烦就大了。"

---

### 5. 引用权威标准时必须翻译成通俗语言

❌ 不能直接说：
> "根据 McCabe (1976) 圈复杂度理论"

✅ 应该说：
> "根据一个 1976 年就被工程界验证过的计算方法"

✅ 或者：
> "Google 的工程师团队总结的经验"

---

### 6. 一句话总结的生成原则

reporter 调用本 Skill 生成总结时，根据数据特征选择表达方式，不套固定模板。

**开头句式不固定**，根据最突出的情况选择：
- 总体改善明显时："这次重构让代码扎实了不少，整体提升了 X%。"
- 某项问题极端时："重构后最值得注意的是……"
- 有维度退步时："整体提升了 X%，但有一个地方需要留意……"
- 全面均衡提升时："六个维度全面改善，整体提升了 X%。"

**具体数字优先**：当 data 字段有具体数字时（如高危漏洞数量、重复率），必须引用，不能用"有几个""减少了"等模糊表述替代。

示例（有具体数字，安全性问题突出）：
> "整体提升了 18%，代码扎实了不少。安全方面进步最明显——之前有 4 个高危漏洞，现在清零了，就像把几扇没锁好的窗户都锁上了。可读性也跟着改善了，代码结构清晰多了。"

示例（改善有限，且有维度退步）：
> "整体小幅提升（+6%），但有一点需要留意：重构后复杂度反而升了 2 分，核心逻辑比之前更绕了——这不一定是坏事，但值得检查一下。"

示例（接近满分，全面提升）：
> "六个维度全面改善，整体提升了 31%。重复代码从 42% 降到了 3%，相当于把重复砌的墙全拆干净了；安全漏洞也从 6 个降到 0。"

示例（进步极小）：
> "这次重构变化不大（+4%），各维度基本持平。如果想继续优化，可维护性（当前 5 分）是提升空间最大的方向。"

---

### 7. 长度控制

- 一句话总结：**最多 3 句话**
- 单个维度解释：**最多 2 句话**
- 改进建议：**每条建议最多 4 句话**
